TP 1 — Installation de VirtualBox et découverte de Linux

VIRTUALISATION

📅 Date : Septembre 2024

🎯 Objectif : Installer VirtualBox, découvrir Linux et préparer la machine virtuelle Xubuntu

VirtualBoxLinuxXubuntuVirtualisationISO
📥 Télécharger le TP

Contexte

Linux est un système d'exploitation open source créé par Linus Torvalds en 1991, distribué sous licence GPL. Nous avons utilisé la distribution Xubuntu 25.10, légère et adaptée à une utilisation en machine virtuelle, sans modifier le système Windows hôte.

Travail réalisé

  • Téléchargement et installation de VirtualBox sur Windows
  • Création d'une machine virtuelle avec les ressources adaptées
  • Montage de l'image ISO Xubuntu 25.10 dans le lecteur virtuel
  • Recherche comparative : VirtualBox, Hyper-V, VMware (éditeur, fonctionnalités, usages)
  • Étude des modes réseau : NAT, Accès par pont, Réseau interne, Host-only

Compétences acquises

  • Comprendre le principe de la virtualisation
  • Différencier les types de virtualisation et leurs usages
  • Configurer une machine virtuelle dans VirtualBox
  • Connaître les distributions Linux et leurs particularités

TP 1 (suite) — Configuration avancée de la machine virtuelle

VIRTUALISATION

📅 Date : Septembre 2024

🎯 Objectif : Explorer les options de configuration réseau et démarrer l'installation

VirtualBoxNATRéseauISO
📥 Télécharger le TP

Travail réalisé

  • Exploration des options de configuration de la machine virtuelle
  • Découverte des modes de carte réseau (NAT, Pont, Réseau interne, Host-only)
  • Insertion du fichier ISO Xubuntu 25.10 dans le lecteur DVD virtuel
  • Démarrage de la machine virtuelle sur le DVD

Modes réseau étudiés

  • NAT — accès Internet via la machine hôte, isolé du réseau local
  • Pont (Bridged) — la VM est visible sur le réseau local comme une vraie machine
  • Réseau interne — communication uniquement entre VMs
  • Host-only — communication entre la VM et l'hôte uniquement

TP 5 — Gestion des utilisateurs Linux

ADMIN

📅 Date : Novembre 2024

🎯 Objectif : Comprendre et gérer les utilisateurs, groupes et droits sous Linux

sudoUtilisateursGroupesUID/GID/etc/sudoers
📥 Télécharger le TP

Commandes utilisées

sudo cat /etc/sudoers # Afficher la config sudo cat /etc/group # Lister les groupes sudo useradd nomutilisateur # Créer un utilisateur sudo passwd nomutilisateur # Définir un mot de passe sudo usermod -aG sudo user # Ajouter au groupe sudo

Fichiers système clés

  • /etc/sudoers — définit qui peut utiliser sudo et avec quels droits
  • /etc/passwd — liste de tous les utilisateurs du système
  • /etc/group — liste des groupes et leurs membres
  • /etc/shadow — mots de passe chiffrés (accès root uniquement)

TP 8 — Serveur Web sécurisé HTTPS + PHP + MariaDB + phpMyAdmin

SÉCURITÉ

📅 Date : Février 2025

🎯 Objectif : Sécuriser Apache avec HTTPS/TLS et déployer une stack LAMP complète

HTTPSTLS/SSLPHPMariaDBphpMyAdminLAMP
📥 Télécharger le TP

HTTP vs HTTPS

  • Confidentialité — données chiffrées, illisibles pour un tiers
  • Intégrité — toute modification invalide la signature numérique
  • Authentification — le certificat SSL/TLS prouve l'identité du serveur

Activation SSL + certificat auto-signé

sudo a2enmod ssl sudo systemctl restart apache2 sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \ -keyout /etc/ssl/private/monsite.key \ -out /etc/ssl/certs/monsite.crt

Installation PHP + MariaDB

sudo apt install php libapache2-mod-php php-mysql sudo apt install mariadb-server sudo mysql_secure_installation
